//Learn what the following git commands do:
 
1) git init: Initializes a new Git repository. If you want to place a project under revision control, 
this is the first command you need to learn.
2) git clone: Creates a copy of an existing Git repository. Cloning is the most common way for
developers to obtain a working copy of a central repository.
3) git diff: Diffing is a function that takes two input data sets and outputs the changes between 
them. git diff is a multi-use Git command that when executed runs a diff function on Git data sources.
4)git reset:To review, git reset is a powerful command that is used to undo local changes to the state 
of a Git repo
5)git rm: The git rm command can be used to remove individual files or a collection of files 
6)git log: Git log is a utility tool to review and read a history of everything that happens to 
a repository
7)git show:git-show is a command line utility that is used to view expanded details on Git objects
 such as blobs, trees, tags, and commits
8)git branch: Git branches are effectively a pointer to a snapshot of your changes.
